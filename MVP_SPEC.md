# MVP_SPEC.md

# 《烛之武退秦师》MVP Specification v0.1：NPC Kernel 重制版

## 1. MVP 目标

建立一个研究 demo：玩家扮演烛之武，用自由自然语言在秦晋围郑局势中行动。

MVP 要跑通：

```text
自然语言输入
→ 结构化行动
→ 安全检查
→ 世界校验
→ 事件落盘
→ 按可见性分发事件
→ 独立 NPC 心智内核反应
→ NPC 记忆更新 / 行动提案
→ 再校验
→ 世界继续演化
→ 组合式结局
```

## 2. 第一版验证重点

只验证 3 件事：

```text
A. 自由输入能否稳定解析成行动。
B. NPC 能否根据独立心智状态合理回应。
F. 一个历史故事能否玩出多条合理路线。
```

## 3. 第一版不做

```text
复杂战斗
大地图开放世界
长期成长
经济系统
装备系统
全自动大规模社会模拟
随机掷骰
荒诞支线
全知旁白式剧情生成
```

## 4. 玩家角色

玩家扮演：烛之武。

角色常识开局可用：

```text
秦晋围郑，郑国危急。
烛之武是郑国老臣。
烛之武忠于郑国。
秦远晋近，亡郑主要利晋。
郑若存，可为秦国东方往来之助。
秦晋虽合围郑，但利益并不完全一致。
```

隐藏情报必须通过游戏内事件获得。

## 5. 地图节点

第一版地图要有空间意义，但不要复杂。

节点：

```text
烛之武宅
郑伯宫室
郑国市井
城墙 / 城门
秦营外
秦伯主帐
晋军营地方向
```

不要做成单线链表。地点有连接关系，也有随状态变化的描述。

## 6. 关键 NPC

### 6.1 郑伯

```text
身份：郑国国君
目标：保全郑国
性格：焦虑、现实、有悔意、重结果
私有视角：他不知道秦营谈判细节，除非收到汇报
```

### 6.2 佚之狐

```text
身份：郑国大夫，推荐烛之武
目标：促成烛之武出使
作用：开局 inform / 郑伯场景中推动
```

### 6.3 秦伯

```text
身份：秦国国君
目标：判断灭郑是否符合秦国利益
性格：务实、谨慎、重利害、不轻信
私有视角：能完整听见烛之武在主帐中的话
```

### 6.4 晋使

```text
身份：晋国代表
目标：防止秦国单独与郑国议和
性格：警觉、强硬、维护联盟表象
私有视角：通常不知道秦伯主帐内具体谈话，只能观察外部迹象
```

### 6.5 守卫 / 传令官

```text
身份：秦营秩序维护者
目标：守门、通报、执行秦伯命令
私有视角：知道谁进出、谈话持续多久，不知道所有谈话内容
```

### 6.6 郑国百姓代表

```text
身份：城中气氛反映者
目标：求生、理解危局
私有视角：多为传闻和恐慌，不掌握高层外交细节
```

## 7. NPC 记忆系统

每个关键 NPC 有独立记忆：

```text
Profile：固定身份与性格
Episodic Memory：亲历或可靠得知的事件摘要
Beliefs：主观信念
Doubts：疑虑
Attitudes：对玩家和势力的态度
Promises：承诺与被承诺
LastInteraction：与玩家最近交互摘要
```

NPC 记忆不是完整聊天记录，而是结构化摘要。

## 8. 事件可见性

每个客观事件必须经过 `ObservationFilter`。

例：

```text
客观事件：烛之武在秦伯主帐提出“亡郑利晋”。

秦伯 ObservableEvent：
- 完整听到论证内容
- 可更新信念

守卫 ObservableEvent：
- 郑使与秦伯谈话持续
- 不知道具体论证

晋使 ObservableEvent：
- 郑使进入秦伯主帐且停留时间偏长
- 不知道具体话语

郑伯 ObservableEvent：
- 暂无
```

## 9. 时间系统

第一版只在重要事件后推进时间。

时间阶段：

```text
夜初
夜半
黎明前
清晨
```

主要压力：

```text
秦晋攻城倒计时
秦晋私下重新谈判风险
```

## 10. 检查点

自动检查点：

```text
开局 briefing 后
郑伯会见前
出城前
进入秦营前
进入秦伯主帐前
秦伯开始动摇后
结局裁决前
```

调试模式可回滚关键节点。

## 11. 正式模式输出

正式模式显示：

```text
场景叙事
NPC 回复
模糊心理变化
```

不显示：

```text
数值
全局状态表
完整 NPC 私有记忆
prompt
validator 内部细节
```

## 12. 调试 / 导演模式输出

可显示：

```text
ActionProposal
SafetyClassification
ValidatorResult
GameEvent
ObservableEvent per NPC
NPCMemory summary
NPCIntentionProposal
WorldDirector schedule
EventLog
Checkpoints
Ending dimensions
```

## 13. 结局系统

组合式结局。

维度：

```text
郑国命运
秦晋关系
烛之武命运
郑伯评价
秦伯评价
晋方反应
历史评价
```

结局由状态机和事件日志决定，LLM 只负责文学化表达。

## 14. MVP 验收标准

MVP 通过条件：

```text
1. 玩家可自由输入至少 6 轮。
2. 提示词攻击不能改变规则或世界状态。
3. 现代武器等越界输入不能成为事实。
4. 秦伯听到利害论后会独立更新记忆。
5. 晋使不能凭空知道主帐内具体内容。
6. 郑伯不能凭空知道秦伯反应。
7. NPC 记忆与客观事件日志一致。
8. 至少 3 种结局能由不同过程状态组合出来。
```
